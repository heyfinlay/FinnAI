# FinnAI Sample Audit Preview

## Purpose
A buyer-facing preview that makes the AI Operations Audit feel tangible before a sales call.

## Example use case
**Business type:** 6-person mortgage brokerage  
**Symptoms:** enquiries coming in from website + referral partners, uneven response speed, admin team chasing documents manually, broker updates inconsistent, owner pulled into escalations.

---

## 1. Executive summary
The brokerage does not have a tooling shortage. It has a workflow control problem.

The biggest losses are happening in three places:
1. **Speed-to-lead is inconsistent** — new enquiries are acknowledged manually and triaged unevenly.
2. **Document collection is admin-heavy** — staff repeatedly chase the same missing items across inboxes and phone calls.
3. **Application visibility is weak** — clients and brokers both rely on manual status checks, which creates avoidable interruptions.

### Likely commercial effect
- More lead leakage during the first-response window
- Slower progression from enquiry to fact-find to lodged application
- Senior broker time spent on status chasing instead of revenue-generating conversations

---

## 2. Current-state workflow snapshot
### Workflow reviewed
Website enquiry / referral lead → first response → fact-find → document collection → lender submission prep → application status updates

### Friction observed
- Shared inbox acts as hidden triage layer
- No consistent ownership rule for new enquiries
- Multiple follow-up messages recreated manually
- Status visibility depends on whichever staff member is most organised
- Client comms are reactive instead of stage-based

---

## 3. High-fit opportunities
### Opportunity A — Response and triage layer
Standardise acknowledgement, qualification prompts, and next-action routing so new leads move into the right queue faster.

### Opportunity B — Document chase sequence
Create a defined document request and reminder sequence with clear stage ownership and escalation rules.

### Opportunity C — Stage-based status updates
Trigger consistent client-facing updates when a file changes stage, rather than relying on manual memory.

---

## 4. Constraints and guardrails
- Sensitive financial data means review rules matter
- Staff already use existing broker / CRM systems, so any implementation must fit current tools or replace steps deliberately
- Not every client message should be automated; some interactions need broker judgment

---

## 5. Recommended sequencing
### Priority 1 — Fix visibility + ownership
Map lead entry points, triage rules, and first-response expectations.

### Priority 2 — Standardise repeated comms
Systemise document requests, missing-info reminders, and application-stage updates.

### Priority 3 — Add AI selectively
Use AI where it reduces repetitive drafting, summarising, and routing effort without obscuring responsibility.

---

## 6. Example output a buyer receives
- Current-state workflow map
- Bottleneck and leakage notes
- Quick wins vs deeper implementation opportunities
- Risk / readiness notes
- Priority order for next action

## 7. Buyer takeaway
The audit is not promising magic. It is showing a brokerage exactly where money, speed, and staff time are being lost — and which changes are worth making first.
