# FinnAI Proof Blocks

Use these on landing pages, outbound pages, proposals, or call decks to add credibility without inventing fake client logos or fake metrics.

## Proof block 1 — What the audit actually shows
**Headline:** See where time is being lost before you build anything.

**Copy:**
The audit maps how work moves today, where it gets delayed, where context is dropped, and which AI opportunities are strong enough to act on. Buyers do not just get an idea list — they get a workflow view, bottleneck diagnosis, and a prioritised next-step sequence.

## Proof block 2 — Mortgage broker example
**Headline:** Example: brokerages often leak value between enquiry, document collection, and application updates.

**Copy:**
A typical brokerage does not need more generic automation. It needs faster lead triage, more consistent document follow-up, and cleaner status communication. The audit makes those handoffs visible before implementation starts.

## Proof block 3 — Accounting / bookkeeping example
**Headline:** Example: firms lose time across onboarding, recurring reminders, and repeated context capture.

**Copy:**
Accounting teams often carry the same information across inboxes, job systems, and client files. The audit identifies where repeated admin can be removed, where communication should be standardised, and where AI is actually safe and useful.

## Proof block 4 — Why buyers trust this format
**Headline:** Diagnostic first. Tools second.

**Copy:**
Most firms already have access to tools. What they lack is a clear operational view of where those tools fit. The audit is built to improve the operating model first, so implementation happens in the right order.
